Room — Entity & Database

📖 Docs

Room is the recommended SQLite abstraction. Define @Entity data classes, @Dao interfaces, and a @Database class. Room generates all the boilerplate at compile time.

Kotlin
📚 Deep Dive

@PrimaryKey(autoGenerate = true): Room auto-increments the ID. Set id = 0 when inserting a new row.

@ColumnInfo(name = "..."): Override the SQLite column name (defaults to the field name).

@Ignore: Exclude a field from the database schema.

@Embedded: Flatten a nested data class into the parent table's columns.

@TypeConverters: Teach Room how to store types it doesn't know (e.g. Date, List<String>, enums) by providing a class with @TypeConverter functions.

Thread safety: Room enforces that database operations run off the main thread. With coroutines, use suspend functions in DAOs — Room switches to a background dispatcher automatically.

DAO & Queries

📖 Docs

DAOs use annotations to generate SQL. Return Flow from queries for live updates. Use suspend for one-shot writes.

Kotlin
📚 Deep Dive

Flow vs suspend: Return Flow<T> from read queries to get a live stream that re-emits whenever the table changes. Use suspend for inserts/updates/deletes.

@Insert onConflict: OnConflictStrategy.REPLACE deletes the old row and inserts the new one. IGNORE skips the insert silently. ABORT (default) throws an exception.

@Query with LIKE: Pass "%\${query}%" for substring search — Room binds it safely, preventing SQL injection.

@Transaction: Annotate a DAO function with @Transaction to run multiple operations atomically. Also required on queries that return @Relation data.

Upsert (Room 2.5+): @Upsert inserts or updates in one step — no need for a manual REPLACE strategy.

DataStore Preferences

📖 Docs

Async, coroutine-based replacement for SharedPreferences. Runs on a background thread. Exposes data as Flow.

Kotlin
📚 Deep Dive

DataStore vs SharedPreferences: SharedPreferences runs synchronously (can block UI) and is not safe for complex concurrent access. DataStore is always async via coroutines and is concurrency-safe.

Keys are typed: intPreferencesKey, stringPreferencesKey, booleanPreferencesKey, floatPreferencesKey, longPreferencesKey, stringSetPreferencesKey.

Default value: If a key doesn't exist, data.map { prefs[KEY] } returns null. Use ?: defaultValue or data.map { prefs[KEY] ?: default }.

Proto DataStore

📖 Docs

Type-safe DataStore using Protocol Buffers. Define schema in a .proto file — no stringly-typed keys. Better for complex structured data.

Kotlin

SharedPreferences

📖 Docs

Legacy key-value store. Synchronous. Avoid for new code — use DataStore instead. Still appears in interviews and older codebases.
