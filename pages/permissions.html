Permissions

Runtime permission requests, rationale dialogs, and handling permanent denial.

Remember: Dangerous permissions must be declared in the Manifest AND requested at runtime (API 23+). Normal permissions only need the Manifest declaration.

After Denied / Rationale

📖 Docs

After a denial, check shouldShowRequestPermissionRationale() to decide whether to show an explanation or send the user to Settings.

Kotlin
Interview tip: shouldShowRationale returns true only after the first denial. After "Don't ask again" it returns false — that's when you redirect to Settings.

Settings Redirect
