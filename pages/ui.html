UI — Jetpack Compose

Layouts, composables, and state management building blocks.

Row

📖 Docs

Places children horizontally. Use horizontalArrangement to space them and verticalAlignment to align on the cross axis. Use Modifier.weight() to distribute remaining space proportionally — just like layout_weight in LinearLayout.

Kotlin
📚 Deep Dive

Arrangement options: Start, End, Center, SpaceBetween, SpaceAround, SpaceEvenly, spacedBy(dp).

RowScope: The content lambda of Row has a RowScope receiver. This unlocks scope-specific modifiers only available inside a Row:

  • Modifier.weight(fraction) — proportional width share of remaining space
  • Modifier.align(Alignment.Vertical) — override per-child vertical alignment (e.g. Alignment.Top, CenterVertically, Bottom)
  • Modifier.alignBy(baseline) — align children by their first text baseline

These modifiers only compile inside a Row { } block — using them outside causes a compile error, which is the entire point of scope-based APIs.

Interview tip: If asked how to push one item to the end of a Row, use Modifier.weight(1f) on a Spacer between the leading and trailing items, or use Arrangement.SpaceBetween.

Box

📖 Docs

Stacks children on top of each other (like FrameLayout). Use contentAlignment for default child alignment, or Modifier.align() per-child.

Kotlin
📚 Deep Dive

Common pattern: Overlaying a loading indicator or badge on top of content.

z-order: Later children draw on top. Use Modifier.zIndex() if you need to override draw order.

BoxWithConstraints: When you need to know the available size at composition time, use BoxWithConstraints to access maxWidth / maxHeight.

Column

📖 Docs

Places children vertically. Mirror of Row — same arrangement/alignment API but on the vertical axis. Modifier.weight() distributes height proportionally among weighted children.

Kotlin
📚 Deep Dive

ColumnScope: The content lambda of Column has a ColumnScope receiver. This unlocks scope-specific modifiers only available inside a Column:

  • Modifier.weight(fraction, fill) — proportional height share of remaining vertical space
  • Modifier.align(Alignment.Horizontal) — override per-child horizontal alignment (e.g. Alignment.Start, Alignment.CenterHorizontally, Alignment.End)
  • Modifier.alignBy(alignmentLine) — align children by a custom HorizontalAlignmentLine

These modifiers only compile inside a Column { } block — using them outside causes a compile error, giving you build-time safety instead of a runtime crash.

weight(fill = true vs false): By default weight(1f) tells the child to fill its share. Set fill = false to let the child be smaller than its share (useful for centering within an allocated slot).

Mixed weighted + fixed sizes: Fixed-size children are measured first. Remaining space is split among weighted children proportionally. This is how you build a scrollable body with a pinned footer.

Column with scrollable top + pinned bottom: Give the scrollable area weight(1f) so the pinned footer always sticks to the bottom regardless of content height.

align() vs horizontalAlignment: Column(horizontalAlignment = Alignment.CenterHorizontally) sets the default for all children. Modifier.align() overrides it for one specific child.

Lazy Layouts

📖 Docs

LazyColumn (vertical list), LazyRow (horizontal list), LazyVerticalGrid — only compose and draw visible items.

Kotlin
📚 Deep Dive

keys: Always provide stable keys with items(list, key = { it.id }). This lets Compose avoid recomposing unchanged items during list updates and preserves scroll position.

contentType: When a list has heterogeneous item types, set contentType = { it::class } so Compose can reuse composition nodes efficiently.

LazyListState: Use rememberLazyListState() to programmatically scroll, detect first visible item, or check if scrolled.

Kotlin

Pagers

📖 Docs

HorizontalPager / VerticalPager from androidx.compose.foundation. Pair with TabRow for tab+pager UI.

Kotlin

Image

📖 Docs 🎨 Icon Library

Compose has three image sources: resource painters, vector drawables, and Material icons. For network images use Coil's AsyncImage.

Kotlin
📚 Deep Dive

ContentScale options: Crop, Fit, FillBounds, FillWidth, FillHeight, Inside, None.

Icons.Default vs Icons.Outlined vs Icons.Filled: All come from material-icons-extended. Default = Filled. Always set contentDescription for accessibility.

Coil transforms: Use model = ImageRequest.Builder(context).data(url).crossfade(true).build() for fade-in and other transforms.

Icons

📖 Docs 🎨 Browse All

Material icons ship in two artifacts: material-icons-core (a small essential set) and material-icons-extended (1 000+ icons). Access via Icons.Default.* (= Filled), Icons.Outlined.*, Icons.Rounded.*, Icons.Sharp.*, or Icons.TwoTone.*.

Kotlin
📚 Deep Dive

Dependency: Add implementation("androidx.compose.material:material-icons-extended") for all 1 000+ icons. The core artifact only ships ~150 essentials.

contentDescription: Always provide one for interactive icons (buttons). Pass null for purely decorative icons so TalkBack skips them.

Tinting: Icon applies LocalContentColor.current by default. Override with the tint parameter.

Icon size: Default is 24.dp. Use Modifier.size() to change it.

Text

📖 Docs

Render text with Text(). Center horizontally with textAlign = TextAlign.Center and vertically with Modifier.wrapContentHeight(Alignment.CenterVertically).

Kotlin
📚 Deep Dive

wrapContentHeight(Alignment.CenterVertically): Makes the composable measure at its intrinsic height then position itself centered within the space the parent allocated. Without it, Text placed inside a taller container sits at the top.

textAlign vs Modifier.align: textAlign = TextAlign.Center centers the text within the Text composable's own bounds. Modifier.align() (inside a Box/Row/Column) positions the whole composable within its parent. You often need both.

wrapContentWidth(Alignment.CenterHorizontally): Same idea on the horizontal axis — useful when the parent fills width but the text content is short.

Overflow: TextOverflow.Ellipsis clips long text with "…". Always pair with maxLines.

AnnotatedString: Mix styles inline — buildAnnotatedString { withStyle(SpanStyle(color = ...)) { append("part") } }.

SelectionContainer: Wrap Text in SelectionContainer { } to enable long-press text selection.

Keyboard / IME Animation

📖 Docs

Animate layout when the software keyboard appears/disappears. Hide/show fields based on keyboard state. Use WindowInsets + imePadding() or imeNestedScroll().

Kotlin
📚 Deep Dive

windowSoftInputMode: In the Manifest, set android:windowSoftInputMode="adjustResize" on the Activity (or use WindowCompat.setDecorFitsSystemWindows(window, false) in code) so the window actually resizes and insets work correctly.

imePadding() vs imeNestedScroll(): Use imePadding() to add bottom padding equal to the keyboard height. Use imeNestedScroll() on a scrollable to let the keyboard dismiss by scrolling down.

isImeVisible: WindowInsets.isImeVisible is a Compose state — reading it causes recomposition whenever the keyboard appears/disappears. Ideal for hiding/showing UI elements.

Animated hide/show: Wrap the element you want to hide/show in AnimatedVisibility(visible = !isKeyboardVisible) to get a smooth slide animation.

FocusRequester: Programmatically open the keyboard with focusRequester.requestFocus() + keyboardController.show().

Animations

📖 Docs

Compose animation APIs: AnimatedVisibility, animateFloatAsState, animateDpAsState, animate*AsState, Crossfade, AnimatedContent, and custom transitions.

📚 Deep Dive

EnterTransition / ExitTransition combinators: Use + to combine. E.g. fadeIn() + slideInVertically(). Built-in options:

  • fadeIn() / fadeOut()
  • slideInVertically { it } / slideOutVertically { it } — pass a lambda returning the offset in px (it = full height)
  • slideInHorizontally { -it } — slide in from left
  • expandVertically() / shrinkVertically() — grow/shrink from top or bottom
  • expandHorizontally() / shrinkHorizontally()
  • scaleIn() / scaleOut()

animationSpec: Pass to any animate* function. Common specs:

  • tween(durationMillis = 300, easing = FastOutSlowInEasing) — linear with easing curve
  • spring(dampingRatio = Spring.DampingRatioMediumBouncy) — physics-based, no fixed duration
  • keyframes { durationMillis = 500; 0.5f at 200 } — value at specific keyframe times

updateTransition: For coordinating multiple animated values that share a single state change. Avoids multiple animate*AsState calls that might drift out of sync.

rememberInfiniteTransition: For looping animations (loading shimmer, pulse, breathing effects). Use animateFloat(initialValue, targetValue, infiniteRepeatable(...)).
