sorted / sortedBy

📖 Docs

sorted() uses natural order. sortedBy { } extracts a single Comparable key. Both return a new list — the original is never mutated.

Comparator

📖 Docs

compareBy { } builds a Comparator from a key selector. Pass it to sortedWith(). Chain multiple selectors as varargs for tie-breaking.

Kotlin
📚 Deep Dive

Comparator contract: Returns negative if first < second, zero if equal, positive if first > second. All Kotlin sort functions rely on this.

compareBy vs Comparator { a, b -> }: compareBy is the idiomatic Kotlin way — it extracts keys and uses their natural compareTo. The lambda form gives you full control for unusual ordering.

Performance: Keys extracted by compareBy are evaluated once per element per comparison — not cached. For expensive key extraction (e.g. parsing), pre-map to a Pair first (Schwartzian transform).

Multiple Fields

📖 Docs

Chain comparators with thenBy { } / thenByDescending { } to break ties. Or pass multiple selectors directly to compareBy().

Kotlin
📚 Deep Dive

compareBy varargs: compareBy({ it.lastName }, { it.firstName }) is exactly equivalent to compareBy { it.lastName }.thenBy { it.firstName } — use whichever reads better.

Mixed asc/desc: compareBy only does ascending. Mix with thenByDescending or negate: compareByDescending { it.score }.thenBy { it.name }.

Stable sort: Kotlin's sortedWith uses a stable sort (TimSort) — equal elements keep their original relative order. Safe to chain sorts.

Nulls & Reverse

📖 Docs

Handle nullable fields with nullsFirst() / nullsLast(). Reverse any comparator with reversed() or use sortedByDescending for simple cases.

Custom Comparator

Write a Comparator lambda when ordering doesn't map to a simple key — e.g. priority enums, locale-aware strings, or domain-specific rules.

In-place Sort

📖 Docs

sorted*() returns a new list. sort*() / sortWith() mutates a MutableList in place — useful when you own the list and want to avoid allocation.

Kotlin
