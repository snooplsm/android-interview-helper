Cold flows, hot flows, operators, and best practices for reactive streams in Android.

StateFlow & SharedFlow

📖 Docs

StateFlow — hot flow with a current value, replays last value to new collectors. SharedFlow — hot flow, configurable replay, good for events.

Kotlin
📚 Deep Dive

StateFlow vs LiveData: StateFlow is coroutine-native, works without a lifecycle owner, and is null-safe. LiveData is lifecycle-aware by default. In Compose, always prefer StateFlow.

SharedFlow for events: Use MutableSharedFlow() (no replay) for one-shot UI events (navigate, show snackbar) to avoid re-delivering events after config change.

Equality check: StateFlow uses structural equality — emitting the same value won't trigger collectors. Emit a copy or use SharedFlow if you need every event delivered.

collect & collectLatest

📖 Docs

collect processes every emission. collectLatest cancels the current block if a new value arrives before it finishes.

Kotlin

launchIn

📖 Docs

Terminal operator that launches a coroutine in a given scope to collect the flow. Equivalent to scope.launch { flow.collect { } }.

Kotlin

combine & zip

📖 Docs

combine emits whenever any source updates (using latest from each). zip pairs emissions 1-to-1 and waits for both.

stateIn & shareIn

📖 Docs

Convert a cold Flow into a hot StateFlow or SharedFlow so multiple collectors share a single upstream subscription.

Kotlin
📚 Deep Dive

WhileSubscribed(5000): The 5-second timeout keeps the upstream alive briefly when there are no subscribers — survives config changes without restarting the flow. This is the recommended pattern for ViewModels.

Eagerly vs Lazily: Eagerly starts immediately (always on). Lazily starts on first subscriber and never stops. WhileSubscribed starts/stops with subscribers.
